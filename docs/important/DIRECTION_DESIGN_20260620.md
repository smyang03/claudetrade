## 0. 출발점 — 시스템 손익 방정식 (US, 알파 기준)

```
알파엔진(양수청산 합) +166
   −  드레인(음수청산 합) −152
   ─────────────────────────
   =  순알파(gross)       +14   (거래당 +0.067%)
   −  수수료(206×0.5%)   −103
   ─────────────────────────
   =  net                 −89
```

**한 문장: 검증된 양수알파(+166)를 음수알파 드레인(−152)이 거의 다 상쇄하고, 남은 푼돈(+14)을 수수료(−103)가 먹어 net이 마이너스다.**

이게 모든 방향의 토대다. "엣지가 없다"가 아니라 **"엣지(청산알파)는 진짜인데 드레인(죽은진입)+수수료가 그걸 0 이하로 희석"**한다.

확정 사실(이번 세션):
- 청산 알파는 진짜: TARGET +4.18%/건(100%알파승), CLAUDE_SELL +2.09, 수익청산 전체 +2.31(84%승). β=1 가정이라 상한이지만 β1.5에도 유지.
- 드레인도 진짜 음수알파: LOSS_CAP −2.15%/건(합−107), 시장 탓 아님(베타~0).
- broad 연장 = 베타 신기루(알파 +0.02%/5d). 승자청산은 이미 타이밍 정확(연장 −2~−6%p). 패자 "회복"=손절제거=폭발경로.
- 전략 알파: gap_pullback US −1.09%(승22%, 베타가 가려줌), KR은 claude_price만 양수·Path A 전부 음수.
- selection 무엣지 6x. 집계 gross의 80%가 베타. regime 데이터 죽음(0/757).

## 1. 설계 원칙 — 3개를 분리한다

방향이 흐려지는 이유는 셋을 섞어서다. 항상 분리한다:

| 구분 | 정체 | 다룰 방식 |
|---|---|---|
| **지킬 것** | 청산 알파엔진(+166, 특히 target/claude_sell) | 보존. 건드리지 않는다(연장도 X) |
| **버릴 것** | 음수알파 전략(gap_pullback, KR Path A) | cut. 알파+수수료 동시 회복 |
| **인정할 것** | loss_cap 드레인(−107, claude_price 내부)·수수료·베타 | 못 고침. 줄이거나 받아들임 |

"버리기 = 해결"이 아니다. 버리기는 **가장자리 음수알파 청소**고, 핵심 드레인은 지킬 것(claude_price) 안에 있다.

## 2. 방향 — 5개 기둥 (각 장단점 + 보완)

### 기둥 A. 알파엔진 보존 (최우선, 방어)

- **무엇**: target/claude_sell/profit_ladder 등 재량청산 경로를 보호계약으로 동결. 연장·익절지연·트리거 변경 금지.
- **근거**: +166 알파의 원천. 시장조정 후도 +4.18%/건. 시스템의 유일한 검증된 자산.
- **장점**: 0 비용, 0 리스크. 가장 확실한 가치.
- **단점**: 순수 방어라 **상방이 없다.** 엔진을 "더 키울" 여지가 작다(연장 실패가 증명).
- **단점 보완**: 키우는 길은 "엔진 자체 튜닝"이 아니라 **"엔진에 도달하는 포지션 비율↑"**(기둥 B/C로 우회). 엔진은 그대로 두고 입력을 정리.

### 기둥 B. 음수알파 전략 cut (housekeeping, 명확)

- **무엇**: gap_pullback(US+KR), KR momentum, KR opening_range 정리. **KR은 claude_price 전용으로 좁힘.**
- **근거**: gap_pullback US 알파 −1.09%(승22%), KR Path A 전부 음수. 베타가 raw 손실을 가려줬을 뿐.
- **장점**: cut 1건 = 음수알파 + 수수료 **동시** 회복(예: gap US 22건 cut → net +35 개선). 변별력·shadow 불필요한 가장 확실한 개선.
- **단점**: ① 효과 규모가 제한적(−40 알파 청소 vs 핵심 −107은 claude_price 내부). ② 표본 작은 전략(ORB US +1.27 알파 n7)은 cut하면 작은 양수도 잃음. ③ KR claude_price만 남기면 KR 거래량 급감(기회 축소).
- **단점 보완**: ① cut을 "해결"로 과대평가 않기(기둥 C/D 병행). ② n<10 양수알파 전략은 cut 아니라 **동결 관찰**(축소만). ③ KR claude_price 단독 생존 여부를 클린 데이터로 별도 판정(미달 시 KR 전면 중단까지 열어둠).

### 기둥 C. 드레인 축소 — loss_cap 죽은진입 (핵심, 어려움)

- **무엇**: claude_price 내부의 loss_cap(−107) 비율을 낮추기.
- **시도 가능 옵션과 각 한계**:
  - C1 진입 변별 개선 → **사망**(selection 6x). 불가.
  - C2 진입 후 약한놈 조기컷(weak_mfe) → **실패**(알파 −1.26, 정상 손실HOLD까지 잘림). OFF 유지.
  - C3 진입 빈도↓(claude_price 신규진입 줄이기) → loss_cap 수 비례 감소. **단 target도 비례 감소**(비율 불변).
  - C4 진입 sizing을 작게(probe) 시작 → 드레인 손실폭↓. 단 진입 conviction 무엣지라 승자도 작아짐.
- **장점**: 성공하면 가장 큰 레버(−107). 
- **단점**: ex-ante·mid-flight 둘 다 막혔다. C3/C4는 드레인과 알파를 **같은 비율로** 줄여 net 비율을 못 바꿈.
- **단점 보완**: 비율을 바꾸는 유일한 정직한 길 = **"진입 후 드러난 행동으로 손실폭만 비대칭 축소"**. 즉 승자는 그대로, 죽은놈은 손실을 더 얕게. weak_mfe가 실패한 건 "청산 시점"이 틀려서지 방향이 틀린 게 아닐 수 있음 → **클린 데이터(6/19~)로 "loss_cap 직전 죽은놈" feature를 재측정**(설계서 EXIT_EXTENSION §8 feature 재사용, 단 "컷"이 아니라 "손실폭 제한·재진입금지"로). **미검증이라 shadow부터.** 안 되면 C는 "인정"으로 강등.

### 기둥 D. 수수료/베타 — 비용과 자기기만 (정직성)

- **D1 수수료**: US 손익분기 fee 0.33%(현 0.5%). 우대 등록·실효비용(환전+최소수수료) 측정.
  - 장점: 순수 비용, 0 리스크. 성공 시 +35 규모.
  - 단점: 우리 통제 밖(broker 우대), 소액주문 최소수수료 floor에 걸림, 엣지 아님(숨통).
  - 보완: **먼저 실제 all-in 비용 측정**(가정 0.5% 검증) → 우대 가능분만 현실 반영. 주문 크기↑는 리스크집중이라 신중.
- **D2 베타 자기기만 차단**: 집계 gross·좋은 달·1일+ 보유는 베타 비중 큼. **알파 기준 스코어카드로 전환.**
  - 장점: 잘못된 자축·잘못된 결정 방지. 모든 기둥의 판정을 정직하게.
  - 단점: "진척 느낌"이 줄어 사기 저하 가능.
  - 보완: 리포트를 **벤치마크상대(알파)**로 고정(`benchmark_compare`/QQQ 조정). capture_net_review에 알파열 추가는 후속.

### 기둥 E. 결정 프레임 — 클린 baseline + bar + kill (메타)

- **무엇**: 6/19를 클린 측정 시작점으로. 정량 기준·기한·kill 조건을 **지금** 못 박음.
- **잠정 bar(협의)**: 클린 4~6주, 알파 기준 net(수수료 후) ≥ 0이 연속 or 월 2/3 양수 & 단일 버그청산 비지배. KR claude_price 단독 net 별도 판정.
- **장점**: "아직 모르겠다"를 N주 뒤 "안다"로. 무한 희망/절망 반복 차단(CLAUDE.md 원칙).
- **단점**: 인내 필요, 분석마비 위험.
- **단점 보완**: bar를 **숫자로 미리 고정** + 중간 점검은 `/capture`·integrity_check만(새 분석 남발 금지).

## 3. 현실적 천장 (정직한 한계)

모든 기둥을 완벽히 실행해도:
- 알파엔진 +166 보존 + 드레인 cut 일부(−40) + 수수료 우대(+35) ≈ **net을 −89 → 대략 break-even~소폭+** 로 끌 수 있음(거친 추정, 오염기간).
- **강한 수익 엔진이 되지는 않는다.** 핵심 드레인(claude_price 내부 loss_cap −107)은 selection 죽음으로 못 고침.

→ **천장 = "비용 이긴 얇은 양수 net".** 이걸 알고 가야 함. 그 다음 질문은 운영자 몫: **"얇은 양수 net"이 라이브 자동매매의 운영 리스크를 감수할 가치가 있나** → 기둥 E의 kill-or-scale 결정.

## 5. 한 장 요약

- **지킨다**: 청산 알파엔진(+166). 건드리지 마(연장도 X).
- **버린다**: gap_pullback·KR Path A(음수알파). KR=claude_price 전용.
- **인정한다**: loss_cap 드레인(−107)·수수료·베타. 줄이거나 받아들임.
- **측정한다**: 알파 기준으로(베타 자축 금지), 6/19부터 클린.
- **결정한다**: 기한·bar·kill. 천장은 "비용 이긴 얇은 양수", 그 이상은 없다고 보고 간다.
