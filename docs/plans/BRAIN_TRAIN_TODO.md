# Brain Train TODO

브레인 학습용 별도 모드는 당장 적용하지 않고, 향후 점검 항목으로 보류한다.

## 목적
- 거래 수가 너무 적을 때 학습 데이터 수집 속도를 높일지 검토
- 운영 모드와 실험 모드를 분리할지 판단
- Brain 학습에 들어가는 데이터 품질과 가중치를 구분할지 검토

## 검토 배경
- 국장/미장 모두 신호가 적은 날이 반복됨
- 거래 수가 적으면 `entry/exit/failure` 학습 샘플이 충분히 쌓이지 않음
- 반대로 조건을 무작정 완화하면 저품질 거래가 늘어 Brain 학습을 오염시킬 수 있음

## 검토 질문
- 별도 모드가 정말 필요한가
- 기존 운영 모드를 소폭 완화하는 것으로 충분한가
- 학습용 거래를 일반 거래와 같은 가중치로 Brain에 넣어도 되는가
- 학습용 거래는 별도 태그와 낮은 가중치를 둬야 하는가

## 후보 이름
- `BRAIN_TRAIN`
- `LEARNING_MODE`
- `DATA_COLLECTION`

## 설계 시 확인할 항목
- 주문 크기
  - 기본 주문 대비 몇 %로 줄일지
- 손절/익절
  - 손절을 더 짧게 가져갈지
  - 익절/보유기간을 더 짧게 가져갈지
- 일일 거래 수 제한
  - 과도한 샘플 수집 방지용 cap 필요 여부
- 전략 조건 완화 수준
  - momentum `vol_mult`
  - volatility_breakout `vol_mult`, `k`
  - mean_reversion 기준 완화 여부
- Brain 반영 방식
  - 일반 운영 데이터와 동일 반영
  - 실험 태그 부여 후 가중치 축소 반영

## 추천 방향
- 바로 적용하지 말고, 먼저 기존 운영 모드에서
  - 국장 모멘텀 버그 수정 반영
  - 미국장 조건이 실제로 너무 빡빡한지 관찰
  - 거래 수가 얼마나 회복되는지 확인
- 그 다음에도 거래 수가 부족하면 `BRAIN_TRAIN` 모드 설계 재검토
