# 시스템 성능 개선 설계 — 무엇을 보았고, 무엇을 고치고, 어디로 갈 것인가

작성일: 2026-06-20
범위: US/KR PathB 자동매매 시스템의 **시스템 자체 성능**(수수료/비용 구조 제외)

## 0. 한 줄 요약

이 시스템은 **윈너를 못 챙겨서가 아니라, 나쁜 진입을 너무 많이 해서 net이 본전**이다.
윈너(TARGET 경로)는 이미 56% capture로 잘 챙긴다. 출혈은 **한 번도 안 오른 진입(loss_cap)** 쪽이다.
따라서 가장 빠른 현금화 레버는 **"나쁜 진입을 작게 시작(scale-in)"** 하나로 거의 수렴한다. 나머지는 부차.
단, 모든 데이터가 강세장(QQQ +11.76%)이고 **승률 43.7%는 엣지가 얇다는 경고**다. 진짜 검증은 약세장.

## 2. 무엇을 개선해야 하나 (문제 정의)

### P1. 나쁜 진입 과다 — 최대 출혈 (loss side)
- loss_cap 43건 -112.8%, MFE +0.39%. 진입의 절반 이상이 틀린다(승률 43.7%).
- **이건 청산 손질로 못 고친다.** 진입을 안 하거나, 작게 하거나, 변별력의 문제.

### P2. 미지근한 윈너 — 조기절단 (win side, 2차)
- PRE_CLOSE(34건 22.6%)·PROFIT_LADDER(27건 12.2%)는 **+수익이지만 capture가 낮다.** TARGET(56%)까지 못 끌고 가고 중간에 끊김.
- 단 **TARGET 자체는 이미 잘 챙김 → 여기는 손대지 않는다.**

### P3. 강세장 beta 의존 — 검증 공백
- 전 데이터가 강세장. 월별 net: 4월 -0.51% / 5월 +0.42% / 6월 -0.44%.
- 강세장에서 43% 승률·net 본전은 **엣지가 얇다는 신호.** 약세장 표본 없음.

### P4. 측정 원장 불완전
- net/capture 숫자의 토대(V2 sync, attribution)가 P0로 미완. 결론의 신뢰도 자체가 흔들림.

### P5. 복잡도 과적재 (메타 문제)
- 이력상 레버를 쌓다 롤백하는 패턴 반복(6-14 Phase2/3 롤백, 6-16 weak_mfe 당일 OFF).
- **덜어내기 우선 원칙(CLAUDE.md)** 위반 위험. 새 레버를 무더기로 더하면 안 된다.

## 3. 이유 (근거 — 왜 이 우선순위인가)

- **왜 진입(P1)이 1순위인가**: 손실 side -117%가 이익 side +134.8%를 거의 상쇄한다. 출혈을 막는 게 윈너를 더 챙기는 것보다 dollar 임팩트가 크고, **선택 변별력을 못 고쳐도** 사이즈로 줄일 수 있다.
- **왜 윈너 보호(P2)는 2차인가**: TARGET이 이미 56% capture. 윈너는 잘 챙기고 있어 한계효용이 낮다. 대상은 미지근한 61건(PRE_CLOSE+LADDER)뿐.
- **왜 측정(P4)이 레버보다 먼저인가**: 새는 원장 위에 시뮬을 지으면 거짓 결론. 18/43이 진짜인지부터.
- **왜 약세월 시뮬(P3)인가**: 강세장 평균은 beta로 부풀려진다. 약세월에서 효과가 나야 진짜 alpha.
- **왜 덜어내기(P5)인가**: 이력이 증명 — 많이 쌓을수록 롤백한다. 시뮬이 이긴 1~2개만 켜는 게 생존 경로.

## 4. 개선 레버 — 채택/보류 정리

### 🟦 운영자 아이디어
- **L1 러너 끌고가기 (hold horizon 연장 / 1~2주 스윙)**: 채택하되 **단독 금지**(giveback 위험, 2026-06-14 ladder 롤백 선례). L2와 결합. 대상은 미지근한 윈너 61건.

### 🟩 Claude 아이디어 — 채택
- **L3 정찰진입 (scale-in)** ⭐ **최우선**: 나쁜 진입을 정찰분(~1/3, 150K)으로 시작 → 첫 30~60분 MFE가 임계 넘으면 증액, 안 넘으면 정찰분만 정리. 손실 베이스를 1/3로. **변별력을 안 고쳐도 net이 산다.** 양쪽(나쁜건 작게/좋은건 크게) 비대칭 개선.
- **L2 분할청산 (scale-out)**: 1/3 익절 + 1/3 trailing + 1/3 러너. L1의 giveback 도박을 분산. 대상은 미지근한 윈너.
- **진입 스로틀**: daily entries 40·고정 450K를 낮춰 나쁜 진입 빈도 자체 축소. 가장 단순한 덜어내기.

### 🟩 Claude 아이디어 — 보류/축소
- **L4 레짐 게이팅**: 풀 분류기 대신 **"약세 신호 시 사이즈/빈도 down"으로 축소**. 강세 beta 약점 방어용. 운영자 확인 필수.
- **L5 앙상블 selection 가중**: 보류(느림·한계효과). `best analyst - consensus gap` 인프라는 존재.
- **L6 loss autopsy + MFE 라벨링**: 런타임 레버 아님 → **측정/분석(Phase 0~1)으로 흡수.**
- **L7 자본회전(weak_mfe)**: 이미 OFF, 주차 유지.

## 5. 최종 설계 방향

### 단계 (덜어낸 버전)

```
Phase 0  측정 신뢰성 확보 (먼저)
         - V2 sync / attribution 확인, 18/43 capture 숫자 검증
         - 새면 여기서 멈추고 원장부터. 거짓 토대 위에 레버 금지.

Phase 1  시뮬 (read-only, 코드 레버 0)
         - tools/tail_capture_sim.py 복제·확장 (tools/swing_capture_sim.py)
         - L3(scale-in) / L1+L2(끌고+분할) / 스로틀을 과거 경로로 시뮬
         - 필수 조건: ① 약세월(4·6월) 슬라이스 분리 ② 오버나이트 갭 시나리오
                      ③ 비용 차감 ④ shrinkage(표본 작음 명시)
         - 레버를 net/capture 개선으로 랭킹. 못 넘는 레버는 폐기.

Phase 2  최상위 1~2개만 적용 (전부 적용 금지)
         - 보통 L3(scale-in)부터. US 한정, shadow→enforce.
         - 보호영역(claude_price/profit_ladder/사이징/리스크) 변경 시 MD 위반 사항 보고.
         - 각 변경: ①무엇 ②왜 ③전/후 상태 + CHANGELOG + 운영자확인 설정값표 갱신.

Option 0 항상 병행 가능 — "안 쌓고 줄이고 모은다"
         - 시뮬이 약하면 이게 정답. 사이즈 down + 깨끗한 데이터 N주 수집 후 재결정.
```

### KR 처리
- 모든 레버는 **`US_` 접두로만** 적용. KR 현행 유지(CLAUDE.md "KR live 확대 금지 / 성과 분리").
- KR은 **시뮬 관측만**. 명확한 net 개선 시에만 별도 안건으로 운영자 승인.
- KR net -58.6%는 청산이 아니라 **진입 변별력 문제** 가능성 — execution 레버로 못 고침. 별도 트랙.

- AUTO_SELL_REVIEW cooldown guard, broker truth fail-closed, sizing reason split: 직접 무관 → 변경 없음.
- profit_ladder floor 상향: 2026-06-14 롤백 선례 → 시뮬에서 러너 희생 0 확인 못 하면 제외.

## 6. 직언 (응원 아님)

- 가장 정직한 결론: **빠른 개선은 "나쁜 진입을 작게(L3 scale-in)" 하나로 거의 수렴**한다. 나머진 부차.
- **강세장 43% 승률은 엣지가 얇다는 경고.** 강세장에서 이 정도면 약세장에선 마이너스일 가능성이 높다.
- 약세월 시뮬에서 L3가 손실을 못 줄이면, "살린다"가 아니라 **"천천히 죽인다"를 인정하는 게 올바른 결정**이다. 살리는 쪽으로만 몰지 않는다.
- 레버를 미리 다 켜지 마라. **시뮬이 이긴 것만, 1~2개만.** 이력이 증명했다 — 많이 쌓을수록 롤백한다.
