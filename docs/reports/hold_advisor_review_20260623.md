## 2. 정상 / 문제 구간 (데이터)

### 정상 작동
- **target_extension HOLD**: 최종실현 +1.05%, 승률 65% (n=20). 추세 살아있는 러너 보유 = 잘 작동.
- 5월 강세장 HOLD 위주 운영은 국면에 부합.

### 문제 / 취약 구간
- **prior 근거 `−8.91%p`가 체리픽**: profit_pullback HOLD 전체는 giveback −5.15%(반납건만 −8.80%). 프롬프트 숫자는 **최악 서브셋 평균**이 "전체가 토해낸다"로 읽히게 들어감 = 과장.
- **같은 모드 최종실현은 +2.2%, 승률 88%**(n=137) = 전 모드 중 최고. 즉 "이익 중 HOLD"는 **대부분 녹색 마감**. 익절 우선 prior가 이 88% 승자를 조기 절단할 위험(capture 손실)이 실재.
- **stop_recovery(손실 HOLD)**: 측정 방법에 따라 **−1.36%/승률5%(path_run 조인, n=21)** vs **+1.22%/승률65%(market-ticker-date 조인, n=46)**로 **부호가 갈림** → 누수 단정 불가(측정 자체가 취약).
- **6/16 SELL 급증의 net 개선 증거 ≈ 0**: post-6/16 재량 SELL(CLOSED_CLAUDE_SELL) 실현 표본 1건뿐. "변별 +25%p"는 50건 가상 A/B일 뿐 라이브 무검증(CLAUDE.md도 명시).

## 3. 데이터 오염 판정

**판정: 절대 수치는 오염·confound 의심, 방향은 비오염으로 재현.**

- prior는 런타임에 mfe/regime를 조회하지 않는 **정적 시스템 프롬프트 문자열**. 직접 런타임 오염은 없음.
- 그러나 도출(6/13 원장 재채점)이 **mfe 96~97% NULL 기간**(6/19 fix 전)과 겹침. giveback("peak−realized")은 peak(=mfe)에 의존 → **−8.91%p 절대값은 오염 의심**.
- `hold_mode` 라벨은 **5월 2162건 / 6월 52건** = prior가 사실상 **5월 강세장(QQQ +11%대) 표본**. "익절 우선"이 강세장 confound 위에 섰다 = 약세장 일반화 위험.
- 방향("이익 중 HOLD가 고점 대비 토해낸다")은 비오염 pnl_pct로도 재현(반납 실재). **수치는 흔들리나 현상은 존재.**

## 4. 토론 — 충돌과 수렴

### 핵심 충돌: 같은 데이터, 반대 처방
- **codex(개선파)**: profit_pullback 최종실현 88% 녹색 마감 → 익절 우선 prior는 **승자 조기절단**. 수치 부풀림+국면confound+무검증 → "최선" 아님. 정직화+조건부화+forward-validation 편입.
- **claude(보호파)**: "−8.91 반납"과 "+0.83 실현"은 **직교**(giveback ≠ 최종손실). 모순 아님 → prior 방향 견고. 표본 21로 청산경로 바꾸는 건 profit_ladder 롤백 전례 = 과잉. 수치 재측정(위생)만, 행동 변경 금지.

### 수렴 (양측 합의)
1. **prior 절대수치(−8.91/−2.36)는 클린 데이터로 재측정해 문구 갱신** — 위생. (codex 제안1 = claude 허용)
2. **weak_mfe 부활 안 함** — stop_recovery 누수 미확정(조인 부호 갈림). 둘 다 기각.
3. **방향(익절 규율) 자체는 보존**하되 라이브 검증 필요 — 강세장 confound라 일반화 미검증.

### 미해결 (사회자 판정 필요)
- prior가 "토해냄 방어"(net+)인가 "승자 조기절단"(net−)인가 → **라이브 표본 0으로 양쪽 다 미입증.**

## 5. 사회자 종합 결론

**"지금이 최선"이라 부를 근거는 없다. 단 방향이 틀렸다는 근거도 없다.** 정확히는 **검증 공백 + 근거 위생 불량** 상태다.

운영자 질문 매핑:
1. **어떤 시점 정상/문제**: 5월 강세장 HOLD·target_extension = 정상. **6/16 prior는 5월 confound·체리픽 수치·라이브 무검증 위에 섰다 = 문제 구간.**
2. **지금 방향 최선인가**: 방향은 보존 가능하나 **최선이라 부를 데이터 없음**(효과 라이브 표본 0).
3. **데이터 오염**: YES — 절대수치(−8.91)가 오염기간+체리픽+5월confound. 방향은 비오염 재현.
4. **hold도 교훈 줘야?**: **YES.** hold advisor·prior는 현재 `lesson_validation.py` **밖**(selection/tuner만 보정). prior는 1회 AB 후 영구 주입, 자기 후속 결과로 재검증 안 됨 = "검증된 것만" 원칙 위반.

## 6. 개선 방향 (우선순위 — 방향 제시, 코드는 승인 후)

| # | 개선 | 근거 | 리스크 | 양측 |
|---|---|---|---|---|
| 1 | **prior 문구 정직화/재측정** (6/19 fix 후 클린 데이터로 −8.91→전체값, "단 88%는 녹색 마감" 명시) | 체리픽+오염 수치 교정, 러너 오절단 인상 완화 | 낮음(문구만, 수익경로라 승인 필요) | **합의** |
| 2 | **hold advisor를 forward-validation에 편입** (profit_guard ON/OFF outcome을 lesson 셀로 축적 → valid 뜨면 유지/롤백) | "hold도 교훈" 답. 미검증 영구주입 종식 | 낮음(관측) | codex 강력, claude 무반대 |
| 3 | **profit_guard 효과 라이브 A/B 수집** (2주, net 기준, 미달 시 토글 false kill) | 효과 라이브 표본 0 → 기한·정량 kill 규율 | 낮음(기존 kill-switch 활용) | **합의** |
| 4 | **국면 조건부화** (익절 우선을 bear/weak·반전 active에 한정, BULL+고점갱신은 러너 HOLD 강제) | 5월 confound 보호, 88% 승자 보존 | 중(행동 변경 → 검증 후) | codex 제안 / claude 신중 |
| 5 | **weak_mfe 재ON** | — | — | **기각**(양측): stop_recovery 누수 미확정, 롤백 전례 |

### 즉시 가치 (낮은 리스크, 합의)
**#1(수치 정직화) + #2(forward-validation 편입) + #3(라이브 A/B kill 규율)** — 셋 다 행동(청산) 변경이 아니라 **근거 위생 + 검증 장치**다. profit_ladder 롤백 같은 함정이 없다.

### 보류 (검증 후)
**#4(국면 조건부화)** — 방향은 맞지만 행동 변경이라 #3의 라이브 A/B 결과를 보고. **#5(weak_mfe) 기각 유지.**

## 7. 한 줄 결론

hold advisor의 **방향**(강세장 익절 규율)은 합리적이나, 그 **근거 수치는 부풀려졌고(체리픽), 오염기간·5월 강세 confound 위에 섰으며, 효과는 라이브로 한 번도 검증된 적 없다.** 그리고 hold advisor는 자기 판단을 forward-validation 받지 않는 유일한 경로다. → **답은 "수치 정직화 + 자기 교훈(forward-validation) 편입 + 라이브 A/B kill 규율"이고, 청산 행동을 지금 바꾸는 건 (weak_mfe 포함) 데이터가 받쳐주지 않는다.**
